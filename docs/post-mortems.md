# 📊 Post-Mortems

This document contains reflections on completed games, focusing on what worked, what didn’t, and lessons learned.

---

## 🚀 Speedy Soucer

### ✅ What went well
- Core gameplay loop is functional and fun
- Project was completed end-to-end
- Learned how to structure a basic Godot project

### ❌ What didn’t go well
- Lack of polish in visual feedback
- Limited sound design
- Difficulty spikes could be smoother

### 📚 Lessons learned
- Small feedback improvements greatly improve game feel
- Planning level progression earlier helps balance
- Iteration is more important than initial implementation
